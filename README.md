# 🔧 GitHub Repo Manager

Веб-панель для массового управления видимостью и удалением репозиториев на GitHub. Позволяет быстро скрыть, открыть или удалить сразу несколько репозиториев с помощью удобного интерфейса с чекбоксами.

![Express](https://img.shields.io/badge/Express-5-blue)
![Octokit](https://img.shields.io/badge/Octokit-5-orange)

## Возможности

- 📋 **Список всех репозиториев** — отображает все репозитории владельца с информацией о языке, описании и статусе (публичный/скрытый/форк)
- 🔒 **Массовое скрытие** — выберите несколько репозиториев и сделайте их приватными одним нажатием
- 🔓 **Массовое открытие** — аналогично для перевода в публичный доступ
- 🗑️ **Удаление репозиториев** — с подтверждением через ввод названия (защита от случайного удаления)
- 🔍 **Поиск и фильтрация** — быстрый поиск по названию/описанию, фильтр по статусу (все/публичные/скрытые)
- ⚠️ **Диагностика токена** — автоматическая проверка прав доступа с подсказками
- 🎨 **Тёмная тема** — интерфейс в стиле GitHub Dark

## Скриншоты

Интерфейс содержит:
- **Панель инструментов** — поиск, фильтры по статусу
- **Панель действий** — выбрать все, скрыть/открыть/удалить выбранные, скрыть/открыть все
- **Список репозиториев** — чекбоксы, названия, описания, языки, бейджи статуса
- **Прогресс-бар** — отображает ход массовых операций

## Установка и запуск

### Предварительные требования

- [Node.js](https://nodejs.org/) 18 или новее
- GitHub Personal Access Token с нужными правами (см. ниже)

### Быстрый старт

```bash
# 1. Клонируйте репозиторий
git clone <repo-url>
cd github-tools

# 2. Установите зависимости
npm install

# 3. Создайте файл .env
cp .env.example .env

# 4. Отредактируйте .env и вставьте ваш GitHub токен
# GITHUB_TOKEN=ghp_ваш_токен_здесь

# 5. Запустите сервер
npm start
```

Откройте в браузере: **http://localhost:3000**

### Создание GitHub токена

1. Перейдите на **[github.com/settings/tokens](https://github.com/settings/tokens)**
2. Нажмите **«Generate new token»** → **«Generate new token (classic)»**
3. Укажите имя токена и выберите права:
   - `repo` — полный доступ к репозиториям (для смены видимости)
   - `delete_repo` — для удаления репозиториев
4. Скопируйте токен и вставьте в файл `.env`

> **Примечание:** для fine-grained токенов (на уровне репозитория) нужны права: *Administration (Read/Write)* и *Visibility (Read/Write)*.

## Как использовать

### Смена видимости

1. Отметьте чекбоксами нужные репозитории (или нажмите «Выбрать все»)
2. Нажмите **«🔒 Скрыть выбранные»** или **«🔓 Открыть выбранные»**
3. Подтвердите действие в модальном окне
4. Дождитесь завершения операции

Для массовых операций используйте кнопки:
- **«🔒 Все скрыть»** — делает все публичные репозитории скрытыми
- **«🔓 Все открыть»** — делает все скрытые репозитории публичными

### Удаление репозиториев

1. Выберите репозитории чекбоксами
2. Нажмите **«🗑️ Удалить выбранные»**
3. В модальном окне введите имя репозитория (или «удалить все» при массовом удалении)
4. Нажмите **«Удалить навсегда»**

> ⚠️ **Удаление необратимо!** Все данные, включая issues, pull requests и wiki, будут удалены навсегда.

### Ограничения GitHub

- **Публичные форки нельзя сделать приватными** — это ограничение платформы GitHub, а не баг инструмента. В интерфейсе такие репозитории помечены бейджем «Форк».

## API

| Метод | Эндпоинт | Описание |
|---|---|---|
| `GET` | `/api/user` | Информация о текущем пользователе + права токена |
| `GET` | `/api/repos` | Список всех репозиториев владельца |
| `PATCH` | `/api/repos/visibility` | Изменить видимость репозиториев |
| `DELETE` | `/api/repos/delete` | Удалить репозитории |

### Примеры запросов

**Изменить видимость:**
```bash
curl -X PATCH http://localhost:3000/api/repos/visibility \
  -H "Content-Type: application/json" \
  -d '{"repos": [{"owner": "user", "repo": "my-repo"}], "visibility": "private"}'
```

**Удалить репозиторий:**
```bash
curl -X DELETE http://localhost:3000/api/repos/delete \
  -H "Content-Type: application/json" \
  -d '{"repos": [{"owner": "user", "repo": "old-repo"}]}'
```
